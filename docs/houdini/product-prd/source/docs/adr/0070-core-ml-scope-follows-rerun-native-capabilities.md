# Core ML scope follows Rerun-native capabilities

Houdini Clone's core ML scope should include only what Rerun can already log, visualize, inspect, replay, or naturally receive through its SDK and examples: model references, inference outputs, tensors, embeddings, segmentation, detections, depth, tracks, images, video, and training-progress recordings from external workflows. SAM-style segmentation can be a reference ML workflow, but we should not build in-graph training controls, fine-tuning orchestration, checkpoint management, model registry behavior, training-dataset workflows, or custom ML viewer infrastructure into the initial node-network product, because most ML work should happen in external pipelines that produce artifacts we inspect through Rerun and the graph.
