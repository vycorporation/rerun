# Asset authoring starts with declarative interfaces

Houdini Clone starts procedural asset authoring with Houdini-like declarative interface elements users can create from the GUI: typed inputs, typed outputs, promoted parameters, parameter groups, labels, stable internal names, versions, asset storage location, help text, defaults, ranges, and basic parameter options. Creating an asset is an explicit operation from a subgraph or graph container, defaults to project asset storage, and allows shared asset library storage to be selected when the user intends reuse. Editing the asset interface should feel as close to Houdini's type properties workflow as practical, while typed breaking changes require a new version, explicit migration, or explicit asset upgrade rather than silently breaking linked instances. Custom viewport handles, complex multiparms, scripting callbacks, and custom compiled operator code are deferred until the asset model and everyday authoring workflow are stable.
