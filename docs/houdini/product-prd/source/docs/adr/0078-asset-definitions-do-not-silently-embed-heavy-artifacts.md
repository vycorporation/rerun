# Asset definitions do not silently embed heavy artifacts

Houdini Clone assets package graph structure, typed interfaces, documentation, dependency declarations, and external artifact references, but they do not silently embed large datasets, model weights, Python environments, recordings, or external analysis files. Project assets may reference project data, shared assets should expose required data as typed inputs or explicit dependency requirements, and portable copies of heavy artifacts happen through separate packaged project, artifact bundle, export, or recording workflows. Those workflows must preview expected size, included files, remaining external references, artifact inclusion choices, available content hashes, dependency requirements, and reproducibility warnings before writing the bundle. Large external artifacts are left as references unless explicitly selected for inclusion, and every bundle writes a manifest with artifact role, original locator, bundled path when copied, size, content hash when feasible, source or output node provenance, and external-reference status. This keeps Houdini-like asset authoring from creating hidden large project state or target-specific coupling.
