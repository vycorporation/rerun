# Node graphs use strong data kinds

Houdini Clone uses strong data kinds for node inputs, node outputs, attributes, parameters, connections, layers, procedural assets, and plugins. Connection compatibility applies to the specific named output and input being wired, including multi-output nodes, nodes with a primary output, nodes inserted onto existing connections, reference input operators that import from named graph outputs, and replacement connections created by reconnect-around deletion. Reference input operators use stable-ID-backed reference target identities under readable node paths and remain visible live one-way graph dependencies subject to data-kind compatibility and acyclic data-flow rules; allowed node renames, moves, and path changes preserve the binding, while target deletion, output removal, or disallowed boundary crossing creates a visible missing reference with no automatic rebinding. Multi-source reference input operators may hold a visible reference target set, but every enabled imported target must satisfy the same typed output contract and coordinate compatibility, preserve source provenance, expose per-target enablement, and express richer merge, union, comparison, or conflict behavior through explicit downstream operators. Disabling a reference target excludes it from the evaluated output while keeping the target reference and provenance visible; disabled coordinate-incompatible targets may remain in the set as repairable intent, while enabled coordinate-incompatible targets produce diagnostics and unavailable output until repaired. Deleting a target removes the reference from the set. Reference consumers should appear in node info and deletion warnings for referenced outputs so users can repair or intentionally break dependencies. Reference input operators do not copy source data, apply hidden coordinate transforms, or allow downstream edits to mutate the referenced source. Incompatible connections should be prevented or clearly invalid, and meaningful conversions, reprojections, or substrate coordinate transforms should happen through explicit conversion or substrate projection operators rather than hidden coercion. The editor may offer assisted conversion or assisted projection when a meaningful repair is available, but the user-confirmed result must contain a normal visible node on the connection or reference path, selected or highlighted briefly after insertion and fully inspectable, editable, movable, bypassable, disableable, and deletable.
