# Asset instances pin exact definitions

Houdini Clone follows Houdini-like asset versioning while prioritizing project reproducibility: asset definitions have stable internal names and versions, new asset instances may use the currently selected or latest compatible definition, and existing instances remain pinned to their exact definition until the user explicitly upgrades or changes them. New asset instances match their pinned definition by default. Allow Editing of Contents unlocks an instance into local editable graph material; Save Asset Definition writes compatible local edits back to the definition; Match Asset Definition discards local unlocked edits and relocks the instance to its pinned definition. Editing an existing definition updates linked matched instances of that same definition only for compatible asset interface changes; breaking changes to typed inputs, outputs, promoted parameters, or saved parameter values require a new asset version, explicit migration, or explicit asset upgrade. External reference input operators cannot target private internals of matched asset instances, because that would couple downstream graphs to implementation details outside the asset interface; unlocked instances may use local internal references while they remain unlocked. Creating a new asset version does not silently migrate old instances.
