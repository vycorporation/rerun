# Rerun targets use Rerun-native mappings

Houdini Clone keeps graph-owned outputs semantic and viewer-agnostic, but Rerun output targets should map requested outputs into Rerun-native archetypes, components, timelines, annotations, and recordings wherever Rerun already has ecosystem concepts for them. Rerun-specific entity paths, timeline choices, viewer API objects, transport/session state, panel state, and adapter internals stay inside the Rerun target adapter rather than leaking into the graph model. Rerun output operators are target-specialized output operators that may express Rerun as the preferred output target and expose Rerun-targeted options through the adapter, but capability negotiation may still require a user-visible target choice if Rerun cannot satisfy the requested fidelity or capability. Rerun should receive viewer-ready output for the requested filtered/styled state rather than being required to hold every hidden source record, while the graph remains the owner of filtered-view meaning, record identity, style provenance, time semantics, capability requirements, and output-target commands. This preserves future output-target flexibility without building a private parallel Rerun schema that fights the viewer we are choosing to use first.
