# Rerun-native graph UI is the next spike

The next graph UX spike should implement the graph, layer, parameter, and orchestration UI in Rerun's native Rust viewer framework rather than SvelteKit, Tauri, Svelte Flow, or another JavaScript-heavy app shell. This avoids a large JavaScript dependency surface, avoids graph-editor library limits, and tests whether the Houdini-like workflow can live close to Rerun's performant computer-vision viewer; the graph model, typed operator contract, and output target contract should remain independent enough to drive other tools later. Rerun-specific concepts belong in the Rerun target adapter and Rerun-native mapping, not in the durable graph model. Rerun's extension interfaces are still a stability risk, so the spike should prefer upstream-compatible viewer extension or embedding first and only move to a maintained fork when a required capability cannot be achieved otherwise.
