# Support navigable subgraphs and promotable assets

Houdini Clone supports Houdini-like graph hierarchy through navigable subgraphs represented by graph containers and readable graph paths. Subnet and asset operations should work like Houdini wherever product constraints allow: selected nodes can be collapsed into a graph container, compatible external connections are rewired through typed inputs and outputs, users can navigate into the internal graph, and a subgraph or graph container can be explicitly turned into a procedural asset. Procedural assets are the reusable, typed, documented, packageable form that a stable subgraph can become, with Houdini-like labels, stable internal names, namespaces or versions, promoted parameters, typed inputs and outputs, dependency declarations, external artifact references, and an explicit asset storage location. Newly created assets default to project asset storage for portability, while shared asset library storage remains selectable in the creation flow for cross-project reuse. New asset instances may use the currently selected or latest compatible definition, and match that definition by default. Existing asset instances remain pinned to their exact asset definition and version until an explicit asset upgrade or definition change; users can unlock an instance for local editing, save compatible local edits back to the definition, or match the definition again to discard local edits. Reference input operators may import from ordinary editable graph and subgraph outputs, with stable identity preserving references across allowed renames and moves. External consumers of matched procedural assets must reference asset boundary outputs rather than private asset internals; Houdini-like internal `OUT_*` nulls can help asset authors define those outputs, and `IN_*` nulls can clarify internal inputs without replacing typed asset inputs. Unlocked asset instances allow local internal references inside the edited instance. Network boxes, collapsed network boxes, graph notes, and node comments remain organization or authoring context until the user explicitly creates a subgraph or asset; graph notes and node comments packaged inside an asset are internal authoring context unless explicitly promoted into asset documentation.
