# Node graph is the source of truth

Houdini Clone exposes both a layer-based interface and a node-graph interface, but all durable dataset operations are represented in the node graph. The layer stack is a user-facing facade over graph outputs so users can work without directly editing nodes while still producing reproducible workflows, reusable procedural assets, and plugin-compatible operations. Layer promotion, layer removal, layer visibility, and layer order are undoable project commands; visibility and order affect presentation only, while layer-facing filter and style edits create or edit graph-backed nodes and may mark downstream outputs stale. Parameter edits are undoable project commands that mark affected downstream outputs stale and follow the selected evaluation mode; they do not silently materialize output data or mutate source artifacts. Node display and template flags are Houdini-like inspection affordances: each node graph or subgraph has one shared graph display context with one primary display flag and optional read-only template-flagged reference outputs, but durable composition, merged editable data, and output participation still come from layer promotion, explicit merge or comparison operators, and explicit output operators rather than preview state. Ordinary graph copy and paste should not duplicate layer promotion or output participation; that requires an explicit presentation-aware duplication command. Graph-editing viewports default to display and template flags from the shared graph display context, while composed-output viewports default to promoted layers or explicit output operators. V1 does not use a separate render flag concept.
