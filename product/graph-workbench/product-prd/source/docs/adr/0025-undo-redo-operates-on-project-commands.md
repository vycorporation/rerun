# Undo redo operates on project commands

Houdini Clone implements undo and redo as project-level commands against the shared project model. High-frequency UI gestures from the layer stack, node graph, parameter panels, asset authoring, and panel layout can be coalesced into one command per completed user intent, such as a drag, resize, graph-note edit, or slider scrub, while discrete edits like delete, rename, connect, bypass, disable, layer promotion, layer removal, layer visibility, and layer order remain separate commands. Layer visibility and order undo affects presentation only; layer-facing filter and style undo affects graph-backed nodes and downstream stale state. History should not depend on frontend DOM, editor implementation details, evaluation results, cached outputs, running work, or other runtime evaluation state. Undo and redo restore graph and project intent; affected outputs then become stale and are reevaluated according to the selected evaluation mode.
