# Graph runtime is viewer-agnostic

Houdini Clone's graph runtime should be viewer-agnostic from the start, producing typed outputs and commands through a stable output target contract rather than being coupled to one viewer. The contract carries graph-owned semantic payloads, durable presentation intent, status, provenance, record identity, time semantics, capability requirements, and output commands; target-specific APIs, entity paths, timelines, UI state, panel state, transport/session handles, and adapter internals remain inside target adapters. Output operators may be generic or target-specialized, and either kind may name a preferred output target, but target-specific options must remain adapter-owned and output targets must use visible capability negotiation rather than silent fallbacks: a target can provide a native mapping, use a declared prepared representation, warn with a lower-fidelity mapping, or mark the output unsupported. When negotiation finds multiple, imperfect, or unsupported targets, users should see the reason and choose whether to use the preferred target with warning, switch target, use a prepared representation, or cancel. Rerun is the first viewer target and driven tool, but the same graph orchestration model should be able to target other viewers, notebooks, renderers, files, services, or computer-vision tools later. Viewport modes should reflect graph-owned intent: graph-editing viewports inspect the shared graph display context's display and template flagged outputs, while composed-output viewports inspect promoted layers or explicit output operator results through the current viewer target.
