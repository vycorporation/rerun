# Default to Houdini-like graph ergonomics

Houdini Clone should emulate Houdini graph ergonomics as closely as practical unless product domain constraints require translation or a deliberate divergence: detailed node information, editable node names, copy/paste and duplicate for nodes or small subgraphs, parameter editing in linked and pinnable parameter panels, a distinct current node within the selected node set, operator families in browsable trees, direct and connection-scoped node creation, insertion of compatible nodes onto existing connections, reconnect-around deletion for simple compatible chains, navigable graph hierarchy, graph-editing viewports that follow one shared graph display context per node graph or subgraph, read-only node template flags for reference overlays, single-output and multi-output nodes whose typed outputs can connect to compatible inputs elsewhere in the graph, optional primary outputs for quick wiring and inspection defaults, Houdini-like null operators as named pass-through anchors with `OUT_*` and `IN_*` naming conventions, visible single-source and multi-source reference input operators for Object-Merge-like imports from compatible graph outputs without hidden per-source coordinate transforms, assisted projection when coordinate-incompatible reference targets can be repaired through explicit visible projection nodes, Houdini-like per-source enablement for multi-source reference target sets, Houdini-like reference navigation through readable paths, node info, badges, search, jump actions, and delete warnings rather than permanent cross-boundary wires by default, explicit output operators instead of a separate render flag, and graph-backed durable operations. Routine subnet, asset, selection, mask, filter, and style operations should follow Houdini behavior by default, translated into the product's typed spatial/CV vocabulary. The product should keep these interaction strengths while avoiding VFX-specific scope.
