# Graph evaluation is demand-driven and work-item-aware

Houdini Clone evaluates node graphs lazily and demand-first: only outputs needed by visible layers, requested node info, exports, or downstream dependencies should compute. Node outputs are cached by evaluation fingerprint, including input identity, reference input targets, reference target sets, reference target enablement, reference coordinate compatibility, parameters, operator version, requested output, and source artifact identity. Reference input operators are live one-way dependencies: when a referenced output changes or a target is enabled or disabled, the referencing output becomes stale and follows normal graph evaluation, while frozen copies require explicit materialized outputs. Enabled coordinate-incompatible reference targets block the affected reference output until repaired or disabled. Long-running work is represented as visible work items that can run in parallel and expose status, progress, cancel, retry, canceled, superseded, failed, and cached states; superseded work may remain as diagnostic history or cache data for the exact old fingerprint, but must not update the current graph output. Retry targets the current graph request by default; rerunning a historical fingerprint is an explicit diagnostic action and cannot update current graph outputs unless the fingerprint still matches.
