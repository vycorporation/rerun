# V1 data flow is acyclic

Node graph evaluation in v1 should use acyclic data-flow connections: users can branch, compare, bypass, disable, group, comment, and organize freely, but connection topology cannot create implicit feedback loops. Normal graph editing should block obvious invalid or cyclic connection attempts, and automatic graph repairs such as reconnect-around deletion must also preserve acyclic data flow. Loaded, migrated, or externally generated graph data may preserve invalid connections as visible diagnostics instead of silently deleting them. This keeps demand-driven evaluation, cache invalidation, node info, layer bindings, and Rerun output mapping predictable while leaving room for explicit future feedback-style operators if a real workflow needs them.
