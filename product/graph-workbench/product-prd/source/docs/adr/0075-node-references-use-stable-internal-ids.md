# Node references use stable internal IDs

Nodes should have Houdini-like user-editable names and readable node paths for readability, graph search, and navigation, but durable graph references should use stable internal node IDs rather than mutable names or paths. Node names should be unique within their parent node graph or subgraph, with duplicate rename, creation, paste, or duplicate attempts rejected or resolved through predictable suffixes. Pasted or duplicated nodes should receive new stable node IDs, while copied editable graph material such as parameters, node comments, user graph styling, layout positions, selected network boxes, graph notes, routing dots, and internal wiring is preserved by default. External connections, cached outputs, running work, current node, selected node set, display/template flags, layer promotion, and output participation should not be silently copied by ordinary paste or duplicate; duplicating those presentation or publication relationships requires an explicit presentation-aware duplication command. The operator type should remain visible separately from the editable node name so users can always tell what operation a renamed node performs. Houdini updates references when nodes are renamed; using stable IDs gives this product the same user-facing safety while protecting connections, layer bindings, asset internals, output operators, diagnostics, and serialized graph data from accidental breakage when names or paths change.
