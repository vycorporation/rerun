# Projects reference external datasets by default

Houdini Clone project files reference external source datasets and large external artifacts by path, URI, or locator rather than embedding large curves, polygons, point clouds, recordings, model weights, or analysis files by default. Projects store graph state, dataset references, cached metadata, styling, viewport state, asset definitions, and small manifests. Portable sharing happens through explicit packaged project or artifact bundle workflows with a packaging preview that shows expected size, included files, external references left behind, artifact inclusion choices, available content hashes, dependency requirements, and reproducibility warnings before copying heavy artifacts. Large external artifacts are left as references unless the user explicitly selects them for inclusion. Every packaged project or artifact bundle should write a bundle manifest recording each artifact's role, original locator, bundled path when copied, size, content hash when feasible, source or output node provenance, and whether the artifact remains external; missing hashes are allowed for streaming or remote artifacts but must appear as reproducibility warnings.
