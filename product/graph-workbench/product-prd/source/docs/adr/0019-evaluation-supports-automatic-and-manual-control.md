# Evaluation uses Houdini-like update modes

Houdini Clone supports three Houdini-like evaluation modes: automatic evaluation, on-interaction-complete evaluation, and manual evaluation. On-interaction-complete evaluation is the v1 default so users can stage rapid parameter, slider, and graph edits without recooking during every intermediate gesture, while still getting feedback after the interaction completes. Stale, canceled, superseded, or failed outputs may continue showing the last successful compatible output with explicit stale, canceled, superseded, or error status; publishing, export, package, and recording operations should warn, block, or request reevaluation rather than silently treating stale displays as fresh results. Reference input operators follow the same mode rules when their referenced outputs change or reference targets are enabled or disabled: they become stale live dependencies rather than copied snapshots. Manual trigger, cancel, and retry controls remain available for long-running nodes and expensive graphs; undo and redo change project state rather than restoring evaluation caches, so affected outputs become stale and follow the current evaluation mode.
